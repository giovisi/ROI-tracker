# Video Homography Pipeline

Adapts [RaghavPu/homography-fitting](https://github.com/RaghavPu/homography-fitting) (single-image court homography + oriented overlay) to work **frame-by-frame on video**, using the pipeline architecture from [enriquedlh97/tennis-virtual-ads](https://github.com/enriquedlh97/tennis-virtual-ads).

---

## What it does

Takes a tennis broadcast video, detects a planar region (court surface, ad banner, etc.) each frame, computes a camera-intrinsics-aware homography that preserves the true physical aspect ratio, and composites an overlay image into that region тАФ with temporal smoothing so the result is stable across frames.

### Two modes of operation

| Mode | Input | SAM2 required? | Script |
|------|-------|----------------|--------|
| **Pre-masked** | Original video + SAM2-masked video (already generated) | No | `run_video_overlay.py` |
| **End-to-end** | Single input video + click prompts | Yes (GPU recommended) | `run_sam2_overlay.py` |

## Usage

### Mode 1: Pre-masked video (no GPU needed)

If you already have a SAM2-masked video (e.g. generated by `video_masker.py` from the homography-fitting repo):

```bash

python scripts/run_video_overlay.py \
    tennis-clip.mp4 \
    masked-output.mp4 \
    result.mp4 \
    --logo sponsor_logo.png

# With debug overlay showing detected corners
python scripts/run_video_overlay.py \
    tennis-clip.mp4 \
    masked-output.mp4 \
    result.mp4 \
    --logo sponsor_logo.png \
    --debug

# Fast iteration: first 200 frames, downscaled
python scripts/run_video_overlay.py \
    tennis-clip.mp4 \
    masked-output.mp4 \
    result.mp4 \
    --logo sponsor_logo.png \
    --max_frames 200 \
    --resize 960x540

# Single image pair (not video)
python scripts/run_video_overlay.py \
    frame0.jpg \
    masked_frame0.jpg \
    output.png \
    --logo logo.png \
    --single
```

### Mode 2: End-to-end with SAM2 (GPU recommended)

No pre-masked video needed тАФ just click where the ad region is:

```bash
# Specify seed point(s) on frame 0
python scripts/run_sam2_overlay.py \
    tennis-clip.mp4 \
    result.mp4 \
    --logo sponsor_logo.png \
    --point 1:500,300

# Multiple seed points for the same object
python scripts/run_sam2_overlay.py \
    tennis-clip.mp4 \
    result.mp4 \
    --logo sponsor_logo.png \
    --point 1:500,300 \
    --point 1:600,350
```

## How it works

### Pipeline stages (per frame)

1. **Mask extraction** тАФ diff the original vs SAM2-masked frame тЖТ binary mask
2. **Hull simplification** тАФ largest contour тЖТ convex hull тЖТ тЙд6 vertices
3. **Vertex classification** тАФ each vertex labeled "internal" or "boundary" (near frame edge)
4. **Corner recovery** тАФ from 2тАУ4 internal vertices, recover full quad via geometric reasoning
5. **Temporal smoothing** тАФ EMA on corner positions with auto-reset on scene cuts
6. **Oriented homography** тАФ decompose H with camera K to get true physical aspect ratio
7. **Compositing** тАФ warp overlay into the quad with correct aspect ratio

### What's new vs the original repo

| Feature | homography-fitting (original) | This pipeline |
|---------|-------------------------------|---------------|
| Input | Single image / first frame | Full video, frame-by-frame |
| Output | Debug images + matplotlib | MP4 video file |
| Temporal stability | N/A | EMA smoothing + cut detection |
| Video I/O | ffmpeg subprocess for 1 frame | cv2.VideoCapture streaming |
| SAM2 integration | Separate script | Integrated end-to-end mode |
| CLI | Minimal | Full flags for tuning |
| Architecture | Monolithic scripts | Modular: geometry / homography / temporal / io |

## Integration with tennis-virtual-ads

This pipeline fills the **CourtCalibrator** and **AdPlacer** roles from the tennis-virtual-ads architecture:

```python
# src/tennis_virtual_ads/pipeline/calibrators/homography_fitting.py

from geometry import extract_mask, get_hull_vertices, classify_vertices, find_corners
from homography import compute_oriented_homography, estimate_camera_matrix
from temporal import CornerStabilizer

class HomographyFittingCalibrator:
    """CourtCalibrator using RaghavPu's hull-based corner detection."""

    def __init__(self, masked_reader, focal_length=None, smoothing=0.3):
        self.masked_reader = masked_reader
        self.focal_length = focal_length
        self.stabilizer = CornerStabilizer(alpha=smoothing)
        self._K = None

    def estimate(self, frame):
        if self._K is None:
            self._K = estimate_camera_matrix(frame.shape, self.focal_length)

        masked_frame = next(self.masked_reader)
        mask = extract_mask(frame, masked_frame)
        pts = get_hull_vertices(mask)
        labels = classify_vertices(pts, mask.shape)
        corners = find_corners(pts, labels)
        corners = self.stabilizer.update(corners)
        homo = compute_oriented_homography(corners, self._K)

        return {
            "H": homo["H"],
            "conf": 1.0,
            "corners": corners,
            "debug": {"aspect": homo["aspect"], "normal": homo["normal"]},
        }
```
